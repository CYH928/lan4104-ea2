## Project Topic: MediSolution

A free, 24/7 AI Medical Assistant Platform for the Hong Kong public.

**Problems addressed:**
- Public A&E wait: 4–8 hours for non-urgent cases
- Private doctor cost: $400–$800+ per visit
- Doctor shortage: 2.1 per 1,000 people (OECD standard: 3.7)
- No reliable 24/7 medical information source

**Target users:** Elderly residents, low-income families, medical students

**Two solutions:**
1. **AI Smart Consultation** — 24/7 AI chatbot; user inputs symptoms or photo; checks Hospital Authority real-time data; outputs risk level (urgent/non-urgent) + preliminary report. Accuracy: 88.6% in pilot test.
2. **Smart Medical Tools** — Smart Map (GPS-based clinic finder), Second Medical Opinion (upload X-ray/CT/MRI for plain-language explanation), Drug Information (photo of medicine → functions, side effects, warnings). Uses HK Dept of Health + Hospital Authority data. Potential reach: 1.4M+ elderly residents.

## A-Grade Script Structure (from Sample Script)

Every script must follow this exact structure:

**Part 1 (Alex Chang):**
- Pain Point 1 — Emotional Appeal: vivid scene ("Picture a typical night in Hong Kong...")
- Pain Point 2 — Logical Appeal: 3 statistics + cause-and-effect + source
- Verbal signpost → Team intro → Mission → Overview → Handover

**Part 2 (Kathy) — for EACH solution:**
- Present: What is it?
- Elaborate: How does it work? (steps) + elaboration technique
- Relatable example (real user story)
- Conclude: measurable results + stakeholder benefits

Required elaboration techniques:
- Solution 1: explaining concepts/features + relatable example
- Solution 2: providing data + implementation details + relatable example

**Part 3 (Alex Ho):**
- Limitation 1 + specific follow-up action
- Limitation 2 + specific follow-up action (must include privacy concern)
- Future development (brief)
- Conclusion — must have all 4 elements: clear signal / restate both solutions / key benefits / vision
- Closing: gratitude + acknowledge judges + invite Q&A

## Important Constraints

- **No tech jargon** — judges are English teachers, not ICT professionals. Avoid: RAG, MedGemma, FastAPI, deployment, encrypted (use "safe and secure"), API, fine-tuning, etc.
- **Commonly known terms are OK:** AI, GPS, X-ray, CT scan, MRI, app, cloud
- **No em-dashes (`—`) in spoken scripts** — speakers cannot say them. Use commas, "and", or split into two sentences instead.
- Word count per speaker: 310–400 words (~2.5 min at HK student speaking pace)
- Scripts are in Markdown; do not convert to code
- Both script files have been grammar-checked and are final. Any edits must maintain spoken-English naturalness (short sentences, no fragments, parallel lists).
